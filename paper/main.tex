\begin{abstract}
Local retrieval-augmented generation tools for scientific reading routinely ship
a hybrid retriever, a reranker, and a ``verifier,'' but almost never report which
of these components actually helps. We treat the question empirically. On two
BEIR benchmarks---SciFact (\NumQueries{} claims, \CorpusSize{} abstracts) and
NFCorpus (\NFqueries{} queries, \NFcorpus{} documents)---we run a strictly
additive ablation with exact search, bootstrap confidence intervals, paired
randomization tests, and Holm correction, entirely on CPU. The dominant factor
is the embedding model: replacing the common \texttt{all-MiniLM-L6-v2} with
\texttt{bge-small-en-v1.5} moves dense retrieval from below BM25 (\SciDenseMini{}
nDCG@10) to clearly above it (\SciDense{}), a paired gain of \SciEmbGain{}
($p{=}\SciEmbP{}$). Reciprocal-rank fusion then beats BM25 on both corpora
($p{=}\SciHybridP{}$ and $p{=}\NFhybridP{}$) and is the best single system on
NFCorpus. We add a confidence-gated corrective re-retrieval that expands and
re-queries only when the first pass is weak, and find it matches plain fusion at
negligible cost while a cross-encoder reranker is far slower and, on these
abstract-level corpora, slightly worse.
Finally, we recast faithfulness verification as evidence detection and find that
a relevance cross-encoder---not a dedicated NLI model---is the strongest cheap
verifier (\FaithCeAuroc{} vs.\ \FaithNliAuroc{} ROC-AUC). Every table is
regenerated on a CPU laptop by three \texttt{make} targets, with no API keys.
Code and data: \url{https://github.com/urme-b/NexusRAG}.
\end{abstract}

\section{Introduction}

The hard part of ``what does the literature say about $X$?'' is finding the few
passages that bear on $X$. Retrieval-augmented generation (RAG)
\citep{lewis2020rag} couples a retriever with a language model so answers are
grounded in retrieved text, and many open-source tools apply this recipe to PDFs
of papers. Two things are usually missing.

First, \emph{the retriever is rarely measured}. A typical project ships a hybrid
retriever---dense embeddings plus BM25, fused and reranked---without ground-truth
relevance judgements, so it is impossible to say which parts help. Second,
\emph{verification is often cosmetic}: systems confirm that a citation marker
such as \texttt{[2]} points to a source that exists, never that the source
\emph{entails} the sentence attached to it.

We take NexusRAG, a local RAG application of exactly this kind, and answer the
questions its original form could not:

\begin{enumerate}
\item \textbf{Which components earn their place?} We build a strictly additive
ablation and evaluate it on SciFact \citep{wadden2020scifact} and NFCorpus, both
via the BEIR \citep{thakur2021beir} judgements, with bootstrap CIs and paired
randomization tests.
\item \textbf{Does a corrective loop help, and at what cost?} We add a
confidence-gated pseudo-relevance-feedback (PRF) pass and compare its
cost/quality against a cross-encoder reranker.
\item \textbf{Can verification be made real?} We replace the citation-index
check with an NLI model and evaluate it as an evidence detector against lexical
and cross-encoder baselines, not a zero-by-construction strawman.
\end{enumerate}

Our contribution is not a new retrieval algorithm. It is an honest,
component-wise measurement that isolates what matters (the embedder, then
fusion), a cheap corrective alternative to reranking, and a content-inspecting
verifier evaluated fairly---all CPU-only and reproducible from one command.

\section{Related Work}

\paragraph{Hybrid retrieval.} Dense retrievers \citep{karpukhin2020dpr,
reimers2019sbert} and lexical scoring \citep{robertson2009bm25} have
complementary failure modes, and fusing their ranked lists with reciprocal rank
fusion (RRF) \citep{cormack2009rrf} is a strong training-free baseline.
Instruction-tuned sentence encoders such as BGE \citep{xiao2024cpack} and E5
\citep{wang2022e5} sharply improved small-model dense retrieval on BEIR; our
study quantifies how much of the usual pipeline that improvement subsumes.
Cross-encoder reranking \citep{nogueira2019passage} re-scores a shortlist with
full query--document attention at high cost.

\paragraph{Corrective and self-reflective RAG.} CRAG \citep{yan2024crag} grades
retrieved passages and triggers corrective actions when retrieval looks poor, and
Self-RAG \citep{asai2024selfrag} trains a model to critique its own retrieval.
Our corrective loop follows this pattern but uses a deterministic, LLM-free
confidence gate and classical pseudo-relevance feedback \citep{lavrenko2001rm},
so the result is reproducible without a generator.

\paragraph{Faithfulness.} RAGAS \citep{es2024ragas} decomposes an answer into
claims and checks each against the context, usually with a large judge model. We
pursue the same goal with a small local NLI cross-encoder \citep{he2021deberta}
and, because SciFact provides sentence-level gold evidence, evaluate the
grounding signal directly as a detector rather than through another model.

\section{The NexusRAG System}

NexusRAG ingests documents (PDF, DOCX, TXT, MD), chunks them, and indexes each
chunk as both a dense vector and a BM25 entry. At query time it retrieves with
each method to a fixed depth, fuses the lists with RRF (smoothing constant
$k{=}\RRFk{}$, the standard value), and---if the top dense similarity is below a
threshold $\tau$---runs a single pseudo-relevance-feedback pass that expands the
query with frequent terms from the first-pass documents and re-retrieves, fusing
the two passes. This is one confidence-gated re-retrieval, not an iterative loop,
and $\tau$ is selected on a held-out split (Section~\ref{sec:setup}), never on the
test set. A local model then
answers using only the retrieved passages with inline citations, and an NLI model
checks that each answer sentence is entailed by its sources. All retrieval uses
exact (brute-force) cosine search, so results are deterministic.

\section{Experimental Setup}
\label{sec:setup}

\paragraph{Data.} SciFact via BEIR is a corpus of \CorpusSize{} abstracts with
\NumQueries{} test claims and abstract-level judgements; NFCorpus is a biomedical
benchmark of \NFcorpus{} documents and \NFqueries{} test queries. For the
faithfulness experiment we use the original SciFact release, which adds
sentence-level evidence labels (\FaithClaims{} dev claims with evidence).

\paragraph{Metrics.} We report nDCG@10, Recall@$\{5,10,20\}$, MRR, and MAP with
binary relevance, each system retrieving to depth 50. We compute 95\% confidence
intervals by bootstrapping over queries ($10{,}000$ resamples) and test each
system against BM25 with a two-sided paired randomization test ($10{,}000$
permutations, add-one estimator), applying a Holm correction across the family of
comparisons. Marked rows ($^{*}$) survive Holm at $\alpha{=}0.05$. All bootstrap
and randomization tests use seed \EvalSeed{}, and all dense retrieval is exact
brute-force cosine search with no approximate-nearest-neighbour index, so the
reported numbers carry no ANN approximation error.

\paragraph{Held-out gate selection.} The corrective gate threshold $\tau$ is
tuned on a held-out split---NFCorpus validation and the SciFact train split---and
then frozen for the test-set evaluation, so the corrective results below are not
fitted to the queries they are scored on.

\paragraph{Models.} The dense model is \texttt{bge-small-en-v1.5} with the
recommended query instruction; the reranker is \texttt{ms-marco-MiniLM-L-6-v2};
the NLI model is \texttt{nli-deberta-v3-small}. Everything runs on CPU; no API
keys are required.

\section{Results}

\subsection{What the components contribute}

Table~\ref{tab:scifact} and Table~\ref{tab:nfcorpus} report the additive ladder;
Figure~\ref{fig:ablation} shows nDCG@10 with bootstrap intervals.

The first row tells most of the story. With the common MiniLM encoder, dense
retrieval (\SciDenseMini{} nDCG@10 on SciFact) is \emph{below} BM25
(\SciBM{})---the well-known reason hybrids are popular. Swapping in BGE lifts
dense retrieval to \SciDense{}, a paired improvement of \SciEmbGain{} nDCG@10
($p{=}\SciEmbP{}$) and now clearly above BM25. The embedder, not the surrounding
machinery, is the dominant factor.

Given a strong dense model, RRF fusion still helps where the two signals are
complementary, but the win over the strongest baseline is real yet modest. We
benchmark fusion against BM25, the strongest lexical baseline, not against
dense-only. On SciFact, hybrid RRF beats BM25 by \SciHybridDelta{} nDCG@10
(95\% paired bootstrap CI \SciHybridDeltaCI{}, $p{=}\SciHybridP{}$) while raising
recall, even though it does not exceed the already-strong dense model on nDCG; on
NFCorpus it is the best single system, beating BM25 by \NFhybridDelta{} nDCG@10
(CI \NFhybridDeltaCI{}, $p{=}\NFhybridP{}$). Both confidence intervals exclude
zero, so the improvement is statistically real but small in magnitude---a
reliable few hundredths of nDCG@10, not a transformative gain. Both fusion
contrasts also survive Holm correction. The last two rungs are deliberately
included as negative results:
query-adaptive fusion weights leave nDCG@10 within $0.001$ of plain fusion on
both corpora, and the corrective pass (Section~\ref{sec:corrective}) is
likewise flat---we keep them in the ladder, labelled as the no-ops they are,
rather than quietly dropping the rungs that did not pan out.

\subsection{The corrective loop: quality at a fraction of the cost}
\label{sec:corrective}

The corrective pass is a single confidence-gated re-retrieval, not an iterative
loop, with the gate threshold $\tau$ fixed on a held-out split before testing.
Because BGE's top-1 similarity is high on nearly all SciFact claims, the gate
rarely fires---at most \SciCorrMaxFire{} of queries even at an aggressive
threshold---so the corrective pass behaves almost identically to plain fusion and
is roughly neutral on nDCG@10 at negligible extra cost (\CorrMs{} vs.\ \BaseMs{}
ms/query). The reranker is the clearer negative result. A cross-encoder reranker
(\texttt{ms-marco-MiniLM}) does not merely fail to help on these abstract-level
corpora---it \emph{hurts}: it lowers nDCG@10 to \RerankND{} (from \BaseND{}) and
Recall@20 to $0.886$ (from $0.900$) while running \RerankSlowdown$\times$ slower
(\RerankMs{} vs.\ \BaseMs{} ms/query; Table~\ref{tab:cost}, timed on the same 120
queries). We report this plainly: on these benchmarks the reranker is both slower
and worse, and the corrective gate, where a strong embedder already retrieves
confidently, simply does not fire. Neither second pass earns its keep here.

\subsection{Faithfulness as evidence detection}

We score every sentence of a claim's cited abstract for whether it is gold
evidence, using NLI relatedness ($1-P(\text{neutral})$), lexical overlap, and a
relevance cross-encoder, and evaluate as threshold-free detection over
\FaithClaims{} dev claims (gold base rate \FaithBaseRate{}).
Table~\ref{tab:faith} reports the result, and it is a useful corrective to the
intuition that an entailment model is the obvious verifier: the relevance
cross-encoder is the strongest detector (\FaithCeAuroc{} ROC-AUC,
$\FaithCeMinusNli{}$ over NLI), with a claim-bootstrap 95\% CI that clears both
baselines. NLI relatedness (\FaithNliAuroc{}, 95\% CI
\FaithNliAurocLo{}--\FaithNliAurocHi{}) is statistically indistinguishable from
lexical overlap (\FaithLexAuroc{})---their intervals almost entirely
overlap---so the dedicated entailment model is weak here: it is only marginally
better than lexical overlap and clearly below the cross-encoder, and we state
that plainly rather than presenting NLI as the verifier of choice. This
evaluation is on SciFact's own claim-verification data
(\texttt{scifact-claims}, dev split), where sentence-level gold evidence makes
the comparison possible.
The lesson is methodological as much as empirical---reported
as detection against baselines that can lose, rather than against a citation
check that scores zero by construction, a content-inspecting signal is a real,
usable measure of support, and the cross-encoder already in the retrieval
pipeline doubles as the best cheap verifier on this task. The ordering is unambiguous on the
threshold-free ROC-AUC and PR-AUC; the F1 column uses a threshold tuned on a
disjoint SciFact train split rather than on the dev set it scores, so it reflects
operating-point performance without peeking. Turning the raw scores into a
calibrated abstention gate is left to future work, since the three scorers'
outputs are on different scales and not directly comparable as confidences.

\section{Discussion}

The practical lesson is an ordering of effort. For local scientific retrieval on
a CPU budget, the first and largest win is a modern small embedder; reciprocal
rank fusion adds a reliable, free improvement, especially on longer documents;
and a cross-encoder reranker buys little for its one-to-two order of magnitude
latency cost on these benchmarks. A cheap confidence-gated re-retrieval is a
better place to spend a second pass than a reranker. For verification, the
distinction that matters is content versus syntax: a content signal---a relevance
cross-encoder, or an NLI model---reported honestly as a detector gives a real
measure of whether a source supports a claim, where a citation check cannot.

\section{Limitations and Reproducibility}

\paragraph{Scope of the wins.} Our retrieval gains, though statistically real,
are small. Hybrid RRF beats the strongest baseline (BM25) by \SciHybridDelta{}
nDCG@10 \SciHybridDeltaCI{} on SciFact and \NFhybridDelta{} \NFhybridDeltaCI{} on
NFCorpus; both 95\% paired bootstrap CIs exclude zero, so the win is genuine but
modest, and we make no claim of a large improvement. The reranker is a negative
result reported honestly: it lowers both nDCG@10 and Recall@20 on these
abstract-level corpora at roughly \RerankSlowdown$\times$ the latency, and the
corrective pass is a single confidence-gated re-retrieval (not an iterative loop)
that is roughly neutral on nDCG@10 here. We do not claim that reranking or
correction helps on these benchmarks.

\paragraph{Datasets and breadth.} We evaluate on only two abstract-level BEIR
datasets---SciFact (\NumQueries{} test claims, the full available set) and
NFCorpus (\NFqueries{} queries). Both are biomedical-leaning, so broader transfer
is untested. Generalization to other BEIR tasks (e.g.\ FiQA, SciDocs), domain
sentence encoders (SPECTER2, SciNCL), additional neural baselines (learned-sparse
SPLADE, ColBERTv2, monoT5), end-to-end answer-quality evaluation (RAGAS or
LLM-as-judge), and full-paper chunking ablations are all future work and are
\emph{not} claimed here. We use binary relevance, so the NFCorpus numbers are not
directly comparable to graded-nDCG leaderboard scores.

\paragraph{Fusion and verifier caveats.} RRF uses the standard constant
$k{=}\RRFk{}$; a sensitivity sweep over $k$ is left to future work. The
faithfulness verifier is validated on SciFact's own claim-verification data
(\texttt{scifact-claims}, dev split) and is weak: the dedicated NLI scorer
(DeBERTa) is only marginally above a lexical-overlap baseline and below a
relevance cross-encoder, so it should not be read as a strong entailment check.
It operates at the sentence level, so multi-sentence reasoning is only partially
captured, and generation quality itself is not scored here---the end-to-end
harness exists (\texttt{nexusrag.eval.generation}) but a small CPU generator is
uninformative for it.

\paragraph{Reproducibility.} All bootstrap and randomization tests use seed
\EvalSeed{}, embeddings are deterministic on CPU, and dense retrieval in the
benchmark is exact brute-force search with no approximate-nearest-neighbour
index, so the numbers carry no ANN approximation error. The BEIR dataset
revisions are pinned (SciFact corpus \texttt{b3b5335}, NFCorpus corpus
\texttt{b5026a0}). The corrective gate threshold $\tau$ is selected on a held-out
split (NFCorpus validation, SciFact train), never on the test set.

\section{Conclusion}

We turned a typical local RAG application into a measured one and found that the
component everyone changes least---the embedder---matters most, while the
components everyone advertises most---reranking and a corrective loop---are, at
this scale, either not worth their cost or best made cheap. We hope the artifact
is a useful template for the many similar systems that ship without numbers.
